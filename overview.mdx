## Introduction

PipeOps runs application and server deployments in your own cloud provider account, so you manage infrastructure without building deployment tooling from scratch.

Use these docs to set up your account, deploy your first application, and improve how you manage and scale your infrastructure over time.

## Using PipeOps

Choose between the web console and the CLI depending on how you prefer to work and automate.

<Callout kind="info">
Use the web console for visual management and day-to-day operations. Use the CLI when you want repeatable automation, scripting, or CI/CD integration. Both connect to the same PipeOps projects and infrastructure.
</Callout>

### Web console

The web console at `console.pipeops.io` gives you a visual way to manage your deployments and infrastructure.

- Visual project and server management
- Real-time monitoring and metrics
- Configuration through forms and wizards
- Team collaboration features

Sign in to the console after creating your PipeOps account to see your projects and deployments.

### Command line interface (CLI)

The PipeOps CLI lets you work from your terminal and integrate deployments into existing automation.

- Fast command-line operations
- Automation and scripting
- CI/CD pipeline integration
- Infrastructure as code workflows

Start with the [CLI overview](cli/overview) to install the CLI and run your first commands.
