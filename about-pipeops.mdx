## About PipeOps

PipeOps is a no-code DevOps platform that helps you build, deploy, configure, scale, and monitor applications without managing complex infrastructure tooling.

<Callout kind="info">

Use this page to understand what PipeOps does at a high level. When you are ready to try it, start with the [Quickstart](/quickstart).

</Callout>

## What we do

PipeOps focuses on four core areas of your application lifecycle: build and deploy, configure, scale, and monitor.

### Build and deploy

Ship code from your repository to a running environment with automated builds and deployments.

- Connect your Git provider, including **GitHub**, **GitLab**, or **Bitbucket**.
- Trigger builds automatically on commits or pull request merges.
- Rely on multi-framework support so you can deploy different stacks from the same account.
- Keep deployment history in one place so you can track what changed and when.

### Configure

Tailor your runtime environment and deployment behavior without writing custom scripts.

- Set server resources and runtime options to match your workload.
- Manage environment variables and secrets per project and environment.
- Adjust deployment settings, such as build commands and health checks, in one place.
- Standardize configuration across teams to reduce drift between environments.

### Scale

Match capacity to demand and keep applications responsive as traffic changes.

- Scale applications up or down based on resource needs and cost targets.
- Deploy to different environments or regions so you can run workloads where you need them.
- Use the same deployment model whether you run test, staging, or production environments.
- Adjust scaling as your product grows without reworking your deployment pipeline.

### Monitor

Stay on top of application health and respond quickly when something goes wrong.

- Track deployment and runtime status from a single dashboard.
- Receive notifications and alerts when deployments fail or applications become unhealthy.
- Monitor key metrics to understand performance and resource usage trends.
- Use monitoring insights to guide scaling and configuration decisions.

## What sets us apart

Teams choose PipeOps when they want DevOps capabilities without maintaining bespoke pipelines and tooling.

- **Simplicity first**  
  Focus on your application, not on wiring together CI/CD, servers, and scripts.

- **Comprehensive lifecycle coverage**  
  Use one platform for build, configuration, scaling, and monitoring instead of juggling separate tools.

- **Flexible and customizable**  
  Adapt environments, resources, and deployment behavior to match your stack and team practices.

- **Streamlined deployment**  
  Go from code to production with predictable workflows and clear feedback at each stage.

- **Cost-aware approach**  
  Right-size resources and environments to control spend while maintaining performance.

## Our commitment to excellence

PipeOps is built to support teams as they grow from first deployment to operating multiple applications at scale.

- We design workflows to be clear, repeatable, and safe for both small and large teams.
- We focus on reliable deployments so you can ship changes with confidence.
- We iterate based on real-world usage and feedback from engineering teams.

<Callout kind="tip">

If you are migrating from homegrown scripts or a manual deployment process, start with a single non-critical service. Use that pilot to refine your conventions, then roll PipeOps out more broadly.

</Callout>
